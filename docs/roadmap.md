# 开发路线图

## 1. 开发原则

优先做 MVP，保证核心闭环跑通。

当前实现状态（代码已完成）：

- Phase 1：项目初始化
- Phase 2：用户与认证
- Phase 3：快乐币与签到
- Phase 4：图片上传与情绪商品
- Phase 5：购买、点赞、收藏、评论
- Phase 6：个人中心数据接口整理
- Phase 7：整体收尾与文档同步

核心闭环：

```txt
注册登录 -> 签到获得快乐币 -> 发布情绪商品 -> 浏览商品 -> 购买商品 -> 作者获得快乐币 -> 点赞收藏评论 -> 个人中心查看记录
```

不要一开始做复杂转卖、盲盒、排行榜、AI 审核或实时行情。

## 2. Phase 1：项目初始化

目标：项目可以启动。

任务：

- 初始化 React + TypeScript + Vite 前端。
- 初始化 FastAPI 后端。
- 配置 PostgreSQL。
- 配置 SQLAlchemy。
- 配置 Alembic。
- 配置基础环境变量。
- 准备容器编排配置。

验收：

- 前端可以启动。
- 后端可以启动。
- 数据库可以连接。
- Alembic 可以执行迁移。

## 3. Phase 2：用户与认证

目标：用户可以注册和登录。

任务：

- 创建 users 表。
- 实现注册接口。
- 实现登录接口。
- 实现 JWT。
- 实现获取当前用户接口。
- 前端实现登录页、注册页。
- 前端保存 token。

验收：

- 用户可以注册。
- 用户注册后默认获得 100 快乐币。
- 用户可以登录。
- 登录后可以获取当前用户信息。

## 4. Phase 3：快乐币与签到

目标：用户可以通过签到获得快乐币。

任务：

- 创建 point_transactions 表。
- 创建 checkins 表。
- 实现 PointService。
- 实现每日签到接口。
- 实现积分余额和流水接口。
- 前端实现签到卡片。
- 前端实现钱包页面。

验收：

- 每个用户每天只能签到一次。
- 签到成功增加 10 快乐币。
- 积分流水正确记录。
- 前端可以看到余额和流水。

## 5. Phase 4：图片上传

目标：用户可以上传图片。

任务：

- 实现本地上传目录。
- 实现图片上传接口。
- 校验图片类型和大小。
- 返回图片 URL。
- 前端实现 ImageUploader。

验收：

- 可以上传 JPG、PNG、WEBP。
- 超过 5MB 的图片被拒绝。
- 上传后前端可预览图片。

## 6. Phase 5：情绪商品

目标：用户可以发布和浏览商品。

任务：

- 创建 emotion_products 表。
- 创建 product_images 表。
- 实现创建商品接口。
- 实现商品列表接口。
- 实现商品详情接口。
- 实现商品修改和删除接口。
- 前端实现首页、市场页、发布页、详情页。

验收：

- 用户可以发布文字和图片商品。
- 发布商品奖励 5 快乐币。
- 商品可以在列表展示。
- 商品详情可以查看。
- 作者可以修改和删除自己的商品。

## 7. Phase 6：购买

目标：用户可以用快乐币购买商品。

任务：

- 创建 purchases 表。
- 实现购买接口。
- 实现购买事务。
- 买家扣除快乐币。
- 作者增加快乐币。
- 写入两条积分流水。
- 更新购买数和快乐指数。
- 前端实现购买按钮和购买状态。

验收：

- 用户不能购买自己的商品。
- 用户不能重复购买同一商品。
- 积分不足不能购买。
- 购买后余额正确变化。
- 作者余额正确增加。
- 积分流水正确记录。

## 8. Phase 7：点赞、收藏、评论

目标：用户可以与商品互动。

任务：

- 创建 product_likes 表。
- 创建 product_collections 表。
- 创建 product_comments 表。
- 实现点赞、取消点赞。
- 实现收藏、取消收藏。
- 实现评论、删除评论。
- 更新商品计数和快乐指数。
- 前端实现相关交互。

验收：

- 同一用户不能重复点赞。
- 同一用户不能重复收藏。
- 评论不能为空。
- 用户只能删除自己的评论。
- 快乐指数正确更新。

## 9. Phase 8：个人中心

目标：用户可以查看自己的记录。

任务：

- 我的主页。
- 我的快乐币余额。
- 我的积分流水。
- 我发布的商品。
- 我购买的快乐。
- 我收藏的快乐。

验收：

- 用户可以完整查看自己的平台行为。
- 页面有 loading、empty、error 状态。

## 10. Phase 9：UI 质感优化

目标：整体达到 Apple-like 的高级、干净、通透质感。

任务：

- 首页 Hero 优化。
- 商品卡片优化。
- 图片展示优化。
- 按钮和表单统一。
- 响应式布局优化。
- 移动端体验优化。
- 空状态和错误状态优化。

验收：

- 页面不拥挤。
- 卡片有统一圆角、阴影、间距。
- 色彩柔和。
- 没有普通后台管理系统味道。
- 没有金融交易所视觉。

## 11. 后续功能

MVP 完成后再考虑：

- 商品转赠。
- 商品转卖。
- 限量商品。
- 快乐盲盒。
- 快乐流通轨迹。
- 成就称号。
- 个性化推荐。
- 管理员审核后台。
- AI 生成标题和标签。
- AI 内容审核。
