# 产品设计文档：快乐交易所

## 1. 产品定位

「快乐交易所」是一个用积分交易快乐内容的轻社交平台。

用户把自己生活里的快乐、治愈、搞笑、鼓励和美好瞬间发布成「情绪商品」。其他用户可以用「快乐币」购买、收藏、点赞、评论这些内容，让快乐从一个人手中流向更多人。

产品不是金融交易平台，而是一个用交易所概念包装的情绪价值社区。

## 2. 产品目标

- 让用户更愿意记录和分享日常快乐。
- 让用户通过购买、收藏、评论获得情绪补给。
- 让内容创作者通过快乐币获得正反馈。
- 通过积分交易和轻社交机制增强趣味性。
- 营造温暖、治愈、积极、安全的社区氛围。

## 3. 用户角色

### 普通用户

可以浏览、购买、点赞、收藏、评论情绪商品。

### 内容发布者

可以发布文字或图片形式的情绪商品，并通过被购买获得快乐币。

### 平台管理员（后续）

可以审核内容、处理举报、管理用户和商品状态。MVP 阶段可以只预留数据结构，不需要完整后台。

## 4. 情绪商品

情绪商品是本平台的核心对象。

每个商品至少包含：

- 标题
- 描述
- 商品类型
- 情绪标签
- 价格
- 图片
- 作者
- 点赞数
- 收藏数
- 评论数
- 购买数
- 快乐指数
- 创建时间

## 5. 商品类型

MVP 商品类型：

| 类型值 | 中文名称 | 示例 |
|---|---|---|
| happy_moment | 快乐的事情 | 今天被陌生人夸了 |
| lucky_today | 今日小确幸 | 买咖啡被送了小饼干 |
| healing_photo | 治愈照片 | 一张阳光洒进房间的照片 |
| beautiful_view | 美丽风景 | 下班路上的晚霞 |
| cute_pet | 可爱小狗/小猫 | 小狗第一次学会握手 |
| funny_joke | 好笑的段子 | 今天朋友说错话的笑话 |
| encouragement | 一句鼓励的话 | 你已经做得很好了 |
| other | 其他 | 无法归类的快乐内容 |

## 7. 核心用户路径

### 发布快乐

1. 用户登录。
2. 进入发布页。
3. 选择商品类型。
4. 上传 1-9 张图片，可选。
5. 填写标题、描述、情绪标签、价格。
6. 发布商品。
7. 系统奖励 5 快乐币。

### 购买快乐

1. 用户进入首页或市场页。
2. 浏览情绪商品卡片。
3. 进入商品详情页。
4. 使用快乐币购买商品。
5. 买家扣除快乐币，作者获得快乐币。
6. 买家查看完整内容。
7. 买家可以点赞、收藏、评论。

### 每日签到

1. 用户进入首页或个人中心。
2. 点击每日签到。
3. 如果今天未签到，获得 10 快乐币。
4. 签到记录写入数据库。

## 8. 积分系统

积分名称：快乐币。

MVP 默认规则：

| 行为 | 积分变化 |
|---|---:|
| 新用户注册 | +100 |
| 每日签到 | +10 |
| 发布情绪商品 | +5 |
| 购买商品 | -商品价格 |
| 商品被购买 | +商品价格 |

所有积分变化必须写入积分流水表。

## 9. 快乐指数

快乐指数用于体现商品受欢迎程度，不做金融化表达。

MVP 计算方式：

```txt
happy_score = purchase_count * 3 + like_count * 1 + collection_count * 2 + comment_count * 2
```

每次购买、点赞、收藏、评论后更新对应计数和快乐指数。

前端可以展示：

- 快乐指数 88
- 被治愈 32 次
- 被收藏 12 次
- 已流通 8 次

不要展示涨跌幅、K 线、收益率等金融化表达。

## 10. 页面设计

### 首页 `/`

目标：像高质感产品官网首屏，而不是普通社区首页。

模块：

- 大 Hero 区域
- 今日快乐补给
- 热门情绪商品
- 最新上架
- 分类入口
- 每日签到卡片

Hero 文案示例：

```txt
让快乐，被看见。
把今天的小确幸，变成可以流通的情绪商品。
```

### 市场页 `/market`

模块：

- 商品卡片网格
- 分类筛选
- 情绪标签筛选
- 价格筛选
- 排序：最新、最热、价格低到高、快乐指数

### 商品详情页 `/products/:id`

模块：

- 大图展示
- 标题
- 作者
- 标签
- 价格
- 快乐指数
- 购买按钮
- 点赞、收藏、评论

未购买状态可以显示基础信息和封面。购买后显示完整描述和所有图片。

### 发布页 `/publish`

模块：

- 类型选择
- 图片上传
- 标题输入
- 描述输入
- 情绪标签选择
- 价格输入
- 发布按钮

### 个人中心 `/me`

模块：

- 用户信息
- 快乐币余额
- 每日签到
- 我发布的商品
- 我购买的快乐
- 我收藏的快乐
- 积分流水入口

## 11. UI 风格

前端 UI 参考 iPhone 官网的配色、质感和样式方向，但不能复制 Apple 官方页面、品牌、图标、素材或具体布局。

设计关键词：

- 高级
- 克制
- 简洁
- 大留白
- 大标题
- 柔和渐变
- 通透玻璃质感
- 圆角卡片
- 轻阴影
- 清晰层级
- 精准排版
- 移动端优先

视觉建议：

- 背景：白色、浅灰、奶油白。
- 辅助色：浅蓝、浅粉、浅紫、浅橙。
- 按钮：黑色、深灰、白色描边或柔和渐变。
- 卡片：大圆角、轻阴影、透明白、细边框。
- 图片：大尺寸、干净展示、避免杂乱边框。
- 字体：系统无衬线字体，标题加粗，正文清晰。

Tailwind 示例：

```txt
bg-neutral-50
text-neutral-950
text-neutral-500
rounded-3xl
bg-white/80
backdrop-blur
border border-white/60
shadow-sm
text-5xl md:text-7xl font-semibold tracking-tight
rounded-full bg-neutral-950 text-white
```
