# 合规与内容安全文档

## 1. 基本原则

快乐交易所是一个积极、治愈、轻松的内容社区。平台的目标是帮助用户分享快乐和获得情绪补给，而不是制造攀比、投机、骚扰或伤害。

本项目不涉及真实金钱交易。所有交易仅使用平台积分「快乐币」。快乐币不能充值、不能提现、不能兑换现金、不能兑换实物资产。

## 2. 禁止的金融化功能

不得实现以下功能：

- 真实支付
- 充值
- 提现
- 现金兑换
- 投资收益
- 收益率计算
- 赌博返利
- 抽奖返现
- 真实金融资产交易
- 虚拟币交易
- NFT
- 区块链交易
- K 线交易系统
- 杠杆、爆仓、做多做空

平台里的“交易”仅指用户用积分获得内容查看、收藏、互动等权限。

## 3. 禁止的内容

平台应禁止以下内容：

- 色情或性暗示内容
- 暴力血腥内容
- 仇恨、歧视、侮辱内容
- 人身攻击
- 恶意羞辱、霸凌
- 侵犯他人隐私的照片或文字
- 未经授权上传他人清晰人像照片
- 泄露个人联系方式、地址、证件号等敏感信息
- 诱导真实金钱交易
- 赌博、返利、刷单、诈骗内容
- 恐吓、威胁、自伤煽动内容
- 明显盗用他人版权作品的内容

## 4. 图片上传安全

图片上传应至少校验：

- 文件类型：JPG、PNG、WEBP。
- 文件大小：默认最大 5MB。
- 文件扩展名和 MIME type。
- 文件保存路径，避免路径穿越。

数据库只保存图片 URL 或路径，不保存图片二进制。

后续建议增加：

- 图片内容审核。
- 图片 EXIF 清理。
- 敏感图片检测。
- 用户举报入口。

## 5. 用户隐私

用户隐私规则：

- 不要公开用户邮箱。
- 不要在前端返回 password_hash。
- 不要在日志中打印密码、token、数据库连接字符串。
- 不要在代码中硬编码 JWT secret。
- 不要泄露用户上传文件的本地真实路径。

## 6. 账户安全

必须实现：

- 密码哈希存储。
- JWT 鉴权。
- 需要登录的接口必须校验当前用户。
- 用户只能修改自己的商品。
- 用户只能删除自己的评论。
- 用户不能直接修改自己的快乐币余额。

## 7. 积分安全

积分是平台内的核心资源，必须谨慎处理。

规则：

- 所有积分变化必须写入 `point_transactions`。
- 不允许直接修改余额而不写流水。
- 购买商品时，买家扣分和作者加分必须在同一事务中完成。
- 积分不足时不能购买。
- 用户不能购买自己的商品。
- 用户不能重复购买同一商品。

## 8. 举报与审核

MVP 阶段至少建议预留举报数据结构。

举报对象：

- 商品
- 评论
- 用户

举报理由：

- 色情低俗
- 暴力血腥
- 侮辱攻击
- 侵犯隐私
- 虚假诈骗
- 诱导真实交易
- 其他

后续可增加管理员审核后台。

## 9. UI 与品牌边界

UI 可以参考 iPhone 官网的高级感、通透质感、留白、排版和柔和渐变，但不得复制：

- Apple Logo
- iPhone 图片
- Apple 官方图标
- Apple 官方素材
- Apple 页面源码
- Apple 特定页面布局
- Apple 品牌文案

正确做法：

- 使用类似的简洁、高级、克制视觉方向。
- 使用自定义插画、渐变、卡片和排版。
- 创建属于快乐交易所自己的品牌语言。

## 10. 社区氛围

产品文案应该温暖、轻松，不要制造压力。

推荐：

- 今天想买一份什么快乐？
- 收到一份小确幸。
- 这份快乐被你收藏了。
- 分享我的快乐。
- 今天也值得被温柔对待。

避免：

- 你的收益落后了。
- 赶紧投资。
- 不买就亏。
- 爆仓。
- 韭菜。
- 收割。
